07 — Lo que Construí · Seguridad e Infraestructura
6 Capas de Seguridad. Todo Protegido.
🔐 Seguridad
🚀 Deploy CI/CD
🏗️ Infraestructura
C1
HTTPS + SSL · Cifrado en tránsito
Todo el tráfico viaja cifrado entre el usuario y el servidor. Ningún dato puede ser interceptado en el camino. Certificado activo en tododrogas.online.
ACTIVO
SSL · TLS 1.3 · Nginx
C2
Acceso al servidor por llave criptográfica
El servidor no acepta contraseñas — solo una llave criptográfica única registrada. Si alguien intenta entrar sin ella, el sistema lo rechaza. No hay contraseña que hackear porque no existe.
ACTIVO
SSH ed25519 · GitHub Secrets
C3
20 credenciales — nunca en el código fuente
Las claves de OpenAI, Microsoft, Meta y Supabase no existen en ningún archivo del repositorio. Se inyectan automáticamente al hacer deploy. Si alguien accede al código, no ve nada sensible.
ACTIVO
Inyección en deploy · CI/CD
C4
Control de acceso por rol en la base de datos
El sistema distingue entre lo que puede ver un paciente, un asesor y un administrador. Cada perfil tiene permisos distintos. Nadie accede a información que no le corresponde, aunque lo intente.
ACTIVO
Supabase RLS · Roles separados
C5
Correo corporativo con autenticación Microsoft
pqrsfd@tododrogas.com.co se conecta mediante un token temporal que Microsoft genera y valida. No existe una contraseña fija — cada conexión usa un permiso que expira automáticamente.
ACTIVO
Microsoft Graph · Azure AD OAuth2
C6
Las claves de la IA nunca llegan al usuario
Cuando Nova TD responde, las claves que la activan permanecen en el servidor. Todas las solicitudes pasan por un intermediario que verifica la identidad antes de ejecutar cualquier acción.
ACTIVO
Proxy seguro · Token validado
